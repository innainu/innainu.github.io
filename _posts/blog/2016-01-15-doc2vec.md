As part of a Cornell Tech project with Chartbeat last semester, I used Python's Scrapy to scrape articles with all its comments off of gizmodo.com. For this blog post, I scraped gizmodo.com for 8,000 articles, trained a doc2vec model, and reduced the resulting vectors to make a neat 2D and 3D visualization. 

Doc2vec, an extension of the word2vec algorithm, is an unsupervised learning method that attempts to learn longer chunks of text (docs) and assign a vector for each document. It's documentation can be seen [here](https://radimrehurek.com/gensim/models/doc2vec.html).

I used each Gizmodo article's text as a different document, and trained the doc2vec model on 8,000 documents. My goal in doing this was to see whether clusters would evolve from the resulting vectors. I was inspired by Christopher Olah's [blog post on MNIST](https://colah.github.io/posts/2014-10-Visualizing-MNIST/) and wanted to see if I could create a similar visualization but with articles.

Doc2vec was successful in detecting similar articles, but the majority of articles did not match each other with such high similarity scores. Most similarity scores were around .5 (similarity scores are measured using cosine similarity, so the closer the score to 1, the closer the 2 article vectors are to each other). As an example, [Drunk Guy Arrested for Kicking a Pepper Robot](http://gizmodo.com/drunk-guy-arrested-for-kicking-a-pepper-robot-1729293319) matched with [You Could Creepily Dress Your Pepper Robot Up Like a Doll](http://gizmodo.com/you-could-creepily-dress-your-pepper-robot-up-like-a-do-1744672027) with a score of .49. Rarely articles would have higher scores like [Nikon D810 Review The Ultimate Adventure Camera](http://indefinitelywild.gizmodo.com/nikon-d810-review-the-ultimate-adventure-camera-1720304051) matching with [Sony A7S Adventure Tested Ultimate Low-Light Mirrorless](http://indefinitelywild.gizmodo.com/sony-a7s-adventure-tested-ultimate-low-light-mirrorles-1726566715) with a score of .77. However, going through both pairs of articles, the scores made a lot of sense! The two robot articles were just about the same topic, whereas the two camera review articles were about the same topic and structured/written identically. 

For dimensionality reduction, I tried many different methods: tSNE, combinations of tSNE and PCA, MDS and isomap, which are all supposed to preserve distances between the original vectors, but the distances between vectors didn't hold in two or three dimensions. Many articles that were supposedly similar in 300 dimensions were not at all similar in the reduced dimensions. I think this is because Gizmodo articles don't form a number of noticeable clusters like MNIST digits, which form 10 very clear clusters. Possibly with a lot more articles, there would be more articles that are highly similar to each other (like the camera review articles) and the dimensionality reductions would be better. I found that tSNE initialized with PCA worked best out of all so I used this for my final visualization.

For the purposes of this visualization, I found the most similar article for each article with their respective similarity score, then only used the articles with the highest 1,000 scores. I found that when using only these articles, the articles actually formed some clusters in two and three dimensions! The visualizations are shown below. To make the visualizations, I adapted code from the talented Christopher Olah's [blog post on MNIST](https://colah.github.io/posts/2014-10-Visualizing-MNIST/). 

Gizmodo articles have meta-tags that include "keywords". I used these keywords to name the clusters.

<script>
var N = 1000;
var D = 2;
var ds_orig2d = new Float32Array(N*N);
var xs = first_1000_xs_2d;
for (var i = 0; i < N; i++)
for (var j = 0; j < N; j++) {
var sum = 0;
for (var d = 0; d < D; d++){
  var diff = xs[D*i + d] - xs[D*j + d];
  sum += diff*diff;
}
ds_orig2d[i + N*j] = Math.sqrt(sum);
}

var D = 3;
var ds_orig3d = new Float32Array(N*N);
var xs = first_1000_xs_3d;
for (var i = 0; i < N; i++)
for (var j = 0; j < N; j++) {
var sum = 0;
for (var d = 0; d < D; d++){
  var diff = xs[D*i + d] - xs[D*j + d];
  sum += diff*diff;
}
ds_orig3d[i + N*j] = Math.sqrt(sum);
}

function knn_points(K, i, D, ds_orig) {
  var knn = [];
  // stupid, dumb, easy hack for testing:
  for (var k = 0; k < K; k++) {
    var x = null, xd = 100000;
    for (var j = 0; j < N; j++) {
      var D = ds_orig[i + N*j];
      if ( i!=j && knn.indexOf(j) == -1 && D < xd) {
        x = j;
        xd = D;
      }
    }
    knn.push(x);
  }
  return knn;
}
